Cdigo.

R Markdown nos permite introducir y correr cdigo de R (y de otros lenguajes de programacin).

Introduccin de Cdigo.

Para introducir cdigo, utilizamos los chunks (pedazos de cdigo). Estos chunks se generan con el siguiente formato:

  • Inicio: Iniciamos un chunk de cdigo con tres ticks (```). Esto le dice a RStudio que a partir de ac active R para procesar la informacin.

  • Nombre del lenguaje de programacin: R Markdown, configurado adecuadamente, puede ejecutar cdigo tanto de R como de otros lenguajes de programacin (como python o julia). Por esto, es necesario especificar el lenguaje de programacin.

  • Opciones: Despus escribimos las opciones que queremos utilizar. Estas opciones son varias y se incluyen para controlar como queremos que se muestre el cdigo y el resultado de este.

  • Nombre: Podemos ponerle el nombre a cada chunk para tener mejor orden del documento. Este es opcional.

  • Cdigo: Aqu es donde va el cdigo que queremos que se ejecute (o que queremos mostrar).

  • Cierre: Volvemos a utilizar los tres ticks para indicarle a R Markdown que vamos a seguir escribiendo texto normal.

Opciones para mostrar o ocultar el cdigo.

Cuando tenemos la facultad de escribir cdigo para nuestros documentos, caemos en una de las siguientes situaciones:

  1. Queremos mostrar el cdigo, pero que no se ejecute (slo queremos ensear cmo se hace algo). En este caso, utilizamos la opcin eval = FALSE dentro de los corchetes.

  2. Queremos mostrar el cdigo y el resultado. Tanto cmo se hace como el resultado final. Este y el anterior son los ideales cuando estamos escribiendo tutoriales de programacin, como este. En este caso no es necesario poner ninguna opcin adicional.

  3. Queremos solo mostrar el resultado (por ejemplo, una visualizacin interactiva) pero no queremos ensear como se hace. Esto es importante si estamos generando un artculo de blog o de peridico digital, ya que muestra el resultado, pero no el proceso para elaborarlo. Para esto, inclumos la opcin echo = FALSE

Opciones para visualizar el contenido resultante.

Para controlar el tamao de una grfica, podemos utilizar la opcin out.width='XXXpx' o out.height='XXXpx', y especificar un tamao en pixeles que se adapte a nuestras necesidades.

Igualmente, para alinear una imagen, podemos utilizar el argumento fig.align y especificar el alineamiento: fig.align='left' alinea la imagen a la izquierda, fig.align='right' a la derecha y fig.align='center' la centra.

Opciones para silenciar mensajes.

Muchas veces R nos manda alertas o mensajes que, si bien son tiles cuando estamos trabajando, puede ser molestos para los lectores o pblico en general. Estos se silencan con las opciones warning=FALSE, cache=FALSE, message=FALSE. Mi recomendacin es utilizarlas todas al mismo tiempo.

Nota: Hay funciones de R (como sf::st_read()) que imprimen mensajes independientemente de escribir estas opciones. En este caso, la impresin de mensaje se controla con el argumento quiet = TRUE de dicha funcin.

Y el cdigo?

El cdigo de R simplemente se agrega dentro de la seccin de cdigo.

Observaciones:

  • Hay que tener en cuenta que, si no silenciamos la impresin del cdigo echo=TRUE, el chunk de cdigo impreso se va a romper cada vez que le pidamos al cdigo que imprima algo.

  • Se recomienda primero hacer un script de R y luego pasar a trabajar en R Markdown, ya que el primer formato nos permite experimentar de manera ms sencilla que el segundo. Sin embargo, si se desea trabajar directo en R Markdown, hay que tener en cuenta que el cdigo se ejecuta con la flecha de play que aparece en la parte superior derecha del chunk, y que el resultado del chunk se imprime justo debajo del chunk, no en la consola, como ocurre normalmente.
